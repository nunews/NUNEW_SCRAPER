# NUNEW Scraper Server

NUNEW 서비스에서 뉴스 콘텐츠의 **본문 데이터 품질을 균일하게 유지하기 위해** 운영되는 보조 크롤링 서버입니다.

일부 언론사는 오픈 API를 통해 기사 메타데이터 또는 요약만 제공하고,  
**전체 본문을 제공하지 않거나 언론사별 포맷 차이가 큰 경우**가 있어  
동일한 사용자 경험을 제공하기 어렵습니다.

본 서버는 이러한 한계를 보완하기 위해  
**오픈 API에서 전체 본문을 제공하지 않는 언론사에 한해**  
웹 사이트에서 기사 본문을 직접 수집하여 반환합니다.

---

## Purpose

- 오픈 뉴스 API의 **본문 미제공 / 부분 제공 문제 보완**
- 언론사별 상이한 데이터 포맷을 정규화
- 서비스 전반의 **기사 본문 품질 및 길이 균일화**
- 프론트엔드 로직 단순화

---

## Features

- 언론사별 기사 본문 선택자 분리 관리
- 한글 인코딩(UTF-8) 안정적 처리
- 오픈 API 미지원 언론사 대상 **선별적 크롤링**
- CORS 지원
- Docker 기반 독립 배포 가능

---

## Supported Press

> 오픈 API에서 전체 본문을 제공하지 않거나,  
> 제공 포맷이 서비스 요구사항에 부합하지 않는 언론사만 포함됩니다.

- 동아일보
- 매일경제
- SBS
- MBN
- 오마이뉴스
- 이투데이

## API

### `GET /scrape`

기사 본문이 오픈 API에서 제공되지 않는 경우,  
언론사 웹 페이지에서 본문을 수집하여 반환합니다.

#### Query Parameters

| name        | type   | description                  |
| ----------- | ------ | ---------------------------- |
| source_name | string | 뉴스 DB에 저장된 언론사 이름 |
| url         | string | 기사 원문 URL                |

#### Example

```http
GET /scrape?source_name=SBS&url=https://news.sbs.co.kr/...
```
